7. Edge Cases & Error Paths

  • Concurrent drag โดย 2 Member: ทั้งคู่ลากการ์ดใบเดียวกันพร้อมกัน — last-write-wins หรือ optimistic lock? (ดู Open Questions ข้อ 4)
  • Task ถูก assign แล้ว: Member อื่นกด assign ซ้อน — override หรือ reject? (FR-4.2 ยังไม่กำหนด)
  • WebSocket disconnect ระหว่าง drag: board state stale — client ควร reconnect และ re-fetch GET /tasks ทั้งหมด
  • Unauthorized transition: MEMBER ลากจาก Todo ไป Done โดยตรง — backend enforce state machine หรือ allow any transition?
  • Board มีการ์ด 200+ ใบ: ต้อง render ลื่น (NFR Performance) — ต้องทดสอบ virtualization หรือ pagination
  • Admin ลบ Task ที่มี assignee: MVP Task หายจาก board — Phase 3 จะส่ง LINE notification แจ้ง assignee

8. Open Questions / Gaps

  • Multi-assignee (OQ-2): 1 Task assign ได้กี่คน — 1 คนหรือหลายคนพร้อมกัน? ส่งผลต่อ data model (assignee_id vs assignees[]) และ UI การ์ด (REQUIREMENTS.md §12 Q2, FR-4.2)
  • Who tests (OQ-3): คอลัมน์ Test ใครเป็น tester? คนทำสามารถเทสงานตัวเองได้ไหม? ส่งผลต่อ permission ของ PATCH /tasks/:id/status ในคอลัมน์ Test (REQUIREMENTS.md §12 Q3)
  • State machine enforcement: Backend enforce valid transitions (เช่น Todo ไป Done ตรงไม่ได้) หรือ allow any-to-any transition ให้ UI จัดการเอง?
  • Concurrent edit resolution: optimistic lock ด้วย version field หรือ last-write-wins? — ส่งผลต่อ WS event payload design
  • Member permissions: MEMBER เปลี่ยน status การ์ดของคนอื่นได้ไหม หรือได้เฉพาะการ์ดที่ assign ตัวเอง?
  • Backend language (OQ-4): Node.js หรือ Python? — ส่งผลต่อ LINE SDK และ WebSocket library ที่ใช้ (REQUIREMENTS.md §12 Q4)
