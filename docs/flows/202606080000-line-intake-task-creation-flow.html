1. Actors & Surfaces

LINE Side

Customer & LINE Platform

  • Customer — พิมพ์ข้อความในกลุ่ม LINE (ไม่ต้องเข้าระบบ)
  • LINE Platform — รับข้อความแล้วส่ง Webhook event ไปยัง Backend
  • Event type: message
  • Header: X-Line-Signature

Backend

Webhook Server & Task Extractor

  • POST /webhook — entry point
  • Verify HMAC-SHA256 signature ก่อนทุก request
  • Task Extractor — keyword /task (MVP) หรือ LLM (Phase 2)
  • LINE Get Profile API — ดึง display name
  • LINE Reply API — ยืนยัน (FR-1.5, optional)

Database & Realtime

Database & WebSocket

  • Table tasks — INSERT สถานะ todo
  • Table users — upsert profile
  • Table line_messages — บันทึก source
  • WebSocket Hub — broadcast task ใหม่ให้ Frontend

7. Edge Cases & Error Paths

  • Invalid X-Line-Signature: return 400 ทันที ไม่ประมวลผล — ป้องกัน request ปลอม (NFR Security)
  • ข้อความไม่ใช่งาน: Task Extractor return empty list → ข้ามโดยไม่มี side effect
  • 1 ข้อความมีหลายงาน: แตกเป็นหลาย Task records ทุกอันผูก source_message_id เดียวกัน (FR-2.2)
  • LINE Get Profile API error: Task ถูกสร้างด้วย line_user_id แต่ display_name ว่าง — behavior retry ยังไม่กำหนด
  • LINE Push message quota หมด: reply confirm ล้มเหลว — Task ยังถูกสร้างปกติ แต่ Customer ไม่ได้รับยืนยัน (FR-1.5 optional)
  • Database INSERT ล้มเหลว: Task หาย — ยังไม่มี retry queue ใน MVP
  • LINE Webhook retry: LINE จะ retry ถ้า server ไม่ได้รับ 2xx — อาจทำให้สร้าง Task ซ้ำ ถ้าไม่มี idempotency key
  • LLM API unavailable (Phase 2): fallback ไป keyword mode หรือ queue ไว้ process ทีหลัง — ยังไม่ตัดสินใจ

8. Open Questions / Gaps

  • Task detection approach (OQ-1): เริ่มด้วย keyword /task (MVP) หรือ AI auto-detect เลย? — ส่งผลต่อ UX ลูกค้า และ complexity ของ Task Extractor (REQUIREMENTS.md §12 Q1)
  • AI provider (Phase 2): Claude API หรือ OpenAI? — ขึ้นอยู่กับ cost และคุณภาพภาษาไทย
  • Idempotency: ถ้า LINE retry Webhook — backend ควรตรวจ messageId ก่อน INSERT เพื่อกัน Task ซ้ำไหม?
  • Multi-task reply: ถ้า 1 ข้อความสร้าง 5 Task — reply confirm รวมเป็น 1 ข้อความหรือแยก?
  • Unknown sender: ถ้า LINE profile API error — แสดง Task ด้วย userId ดิบหรือ "Unknown"?
  • Group scope: ระบบรองรับหลาย LINE Group พร้อมกัน หรือ 1 group ต่อ 1 deployment?
